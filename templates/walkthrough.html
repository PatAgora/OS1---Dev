4 Associate Applies (Behind the Scenes) SYSTEM
What Happens

When an associate submits an application from the portal (see Associate Step 6), the system performs the following automatically:

  • Creates Application record — linked to the Candidate and Job, with initial status "Pipeline"
  • AI Summary auto-generates — Gemini AI reads the associate's CV and produces a plain-text summary of their experience, skills, and suitability
  • AI Match Score auto-generates — Score 0-100 comparing CV to job description, with detailed explanation of scoring rationale
  • Score colour coding: 75+ green (strong match), 50-74 amber (partial match), <50 red (weak match)
Where Staff See It
  • /applications — full applications list with search, filters by engagement/status
  • /workflow — kanban board showing all applications by stage
  • /candidate/<id> — candidate profile with AI Summary card, AI Match Score sidebar card, and all application detail
Application visible in pipeline

9 Start Referencing STAFF
Where

Navigate to /candidate/<id> — scroll to the "Reference Requests" section. Click "Start Referencing".

GATE: The associate must have signed the Employment Reference Declaration (Signable widget on the associate portal) before referencing can begin. If not signed, referencing is blocked.
What Happens When You Click "Start Referencing"
  1. System reads the associate's employment history entries (companies, dates, referee emails, names)
  2. For each entry, creates a ReferenceRequest record with: Company Name, Referee Name, Referee Email, Status ("not_sent")
  3. If no referee email was provided by the associate, the system auto-looks up the company in the Reference Contact Details list (Config C5) and auto-fills the email
  4. The system cross-references each company against the Reference Houses Intel list (Config C5). Flagged companies get an orange warning indicator
  5. If the associate marked "Do not contact" (permission_to_request = No) for a company, the reference is created with "On Hold" status and a red "No permission" badge
  6. References vetting check status → "In Progress"

10 Start Full Vetting STAFF
Where

Navigate to /candidate/<id> — scroll to the "Vetting Checks" section. Click "Start Full Vetting".

What Happens Immediately
  1. An email modal opens instantly — pre-loaded with the Vetting Commencement email template (Config C4)
  2. Staff can review the email, add attachments, and click "Send"
  3. The Verifile API fires in the background — creating screening orders for DBS, employment verification, etc.
  4. All applicable vetting checks are set to status "In Progress"
Email sent: Vetting Commencement email to the associate. Subject and body from the configured template. Can include file attachments (e.g. vetting guidelines document).
Vetting Checks Dashboard

The Vetting Checks section on the candidate profile shows:

  • Required checks banner: Blue info bar listing all checks required by the engagement + job
  • Progress bar: e.g. "4/12 complete" with percentage
  • Bulk assign: Dropdown to assign all checks to a specific analyst
  • Add extra check: Dropdown to add check types not already present
  • Per-check rows: Each check shows type, status badge, assigned analyst, QC status, and action buttons
Vetting Check Statuses
Not Started — check not yet begun In Progress — check underway Complete — check passed QC Complete — quality-checked by a second analyst QC Not Required — no QC needed for this check Referral Approved — exception approved
What the Associate Sees
Associate portal: At /portal/vetting-progress, the associate sees a progress bar (e.g. "4/12 Complete") and an accordion with information about each check type. They also receive direct contact from Verifile for DBS, Right to Work, and Credit Check steps.
Integration: Verifile
Verifile API: DBS screening orders created automatically using the DBS criteria configured on the job (Level, Sector, Purpose, Position, Employer Name). Verifile contacts the associate directly for identity verification documents. Status updates are received automatically and the vetting checks update in real-time.
Vetting must complete before contract
